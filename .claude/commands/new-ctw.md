## CTW 問題仕様 (v3.3)

### 概要
- アカデミックパッセージの空欄に欠けた文字を入力する問題
- 1セットあたり20のターゲットワード、うち10問がユーザー入力（残り10は表示のみ）
- 制限時間: 36分
- CEFR B1-B2レベル

### パッセージ作成ルール
- 200-250語程度のアカデミックパッセージ
- intro（導入1-2文）+ target words（本文）+ conclusion（結論1-2文）の3部構成
- 自然な学術的文章（大学の教科書・講義レベル）
- ターゲットワードを含めて文章が完結すること

### ターゲットワード選定ルール
- 20語のうち10語をユーザー入力（`a` フィールドに欠けた文字を指定）
- 残り10語は表示のみ（`a: null`）
- 入力語の品詞バランス: 名詞3-4、動詞2-3、形容詞2-3、副詞1-2
- 難易度バランス: MOD（やさしめ）5問、HARD（難しめ）5問

### データ形式（変数 `D`）
```javascript
var D = {
  "intro": "導入文（1-2文）",
  "conclusion": "結論文（1-2文）",
  "target": [
    { "w": "完全な単語", "a": "欠けた文字列", "s": "表示される接頭部分", "p": "後続の句読点" },
    { "w": "表示のみ", "a": null, "s": "", "p": "" },
    // ... 計20語
  ]
};
```

#### フィールド説明
- `w`: 完全な単語（正解）
- `a`: ユーザーが入力すべき文字列。`null` の場合は表示のみ
- `s`: 単語の先頭部分（表示済みのヒント）。例: `w: "environment"`, `s: "envir"`, `a: "onment"`
- `p`: 単語直後の句読点（`,` `.` `;` など）。なければ空文字 `""`

#### ヒント（接頭部分）の設計ルール
- MOD問題: 先頭3-5文字を表示（残り2-4文字を入力）
- HARD問題: 先頭1-3文字を表示（残り4-6文字を入力）
- 入力文字数が1文字にならないこと（最低2文字）
- 接頭部分だけで単語が一意に特定できる場合は避ける（複数候補がありえる程度に）

### HTML構造（既存踏襲）
- トップナビ: `Set X of N` / タイトル / タイマー / Review / ユーザーバッジ
- プログレスバー: セット位置を反映
- メインコンテンツ: 中央配置（max-width: 660px）
  - intro文 → ターゲットワード群 → conclusion文
  - `a !== null` の語: 接頭部表示 + 1文字ずつの入力ボックス(`.bx`)
  - `a === null` の語: そのまま表示
- フッター: Menu / Next ボタン
- Reviewオーバーレイ: 各空欄の入力状況

### JS機能（既存踏襲）
- 入力ボックス: 1文字入力で次のボックスに自動フォーカス
- Backspace: 空のボックスで押すと前のボックスに戻る
- セッションストレージ: `ctw_answers_{SET_NUM}` にスコアを保存
- タイマー: 36分カウントダウン（表示/非表示切替可）
- Review: 各空欄のAnswered/Empty状態を表示
- Next: 最終セットなら「解答解説を見る →」、それ以外は次のセットへ

### 解答ページ（ctw-answers.html）
- 既存ページにタブを追加（新セット分）
- 各問題カード:
  - 問題番号（正誤で色分け: 緑/赤）
  - 表示部 + 正解 + 完全な単語
  - 難易度タグ: MOD / HARD
  - 品詞分類（名詞/動詞/形容詞/副詞 等）
  - ユーザーの回答（あれば）
  - 文法パターン・推測ヒントの解説（日本語）

### 解答解説データ形式
```javascript
{
  "pos": "品詞（例: 名詞, 副詞, 形容詞）",
  "d": "難易度（MOD or HARD）",
  "e": "解説文（日本語）— 文法パターンや推測のコツを含む"
}
```

### 検証チェックリスト
- [ ] パッセージが200-250語
- [ ] ターゲットワード20語（入力10 + 表示10）
- [ ] 入力語の品詞バランス: 名詞3-4、動詞2-3、形容詞2-3、副詞1-2
- [ ] 難易度バランス: MOD 5問、HARD 5問
- [ ] 入力文字数が全問2文字以上
- [ ] `s`（接頭部）+ `a`（入力部）= `w`（完全な単語）が全て一致
- [ ] `p`（句読点）が正確に設定されている
- [ ] 過去セットとトピックが重複していない
